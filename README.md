# customLinter

`customLinter` — это кастомный Go-анализатор (линтер) для проверки текстов лог-сообщений.

Проект рассчитан на использование с Go 1.22+ и построен на `golang.org/x/tools/go/analysis`.

## Корневая папка проекта

Корень проекта: `customLinter`.

## Что проверяет линтер

Для логов из `log/slog` и `go.uber.org/zap` реализованы 4 правила:
- сообщение должно начинаться со строчной буквы;
- сообщение должно быть на английском языке;
- сообщение не должно содержать спецсимволы или эмодзи;
- сообщение не должно содержать потенциально чувствительные данные.

## Текущий статус

Этап 1 (базовая структура):
- создана структура проекта;
- настроен `go.mod`;
- реализован базовый анализатор.

Этап 2 (правила):
- реализованы все 4 проверки.

Этап 3 (тестирование):
- написаны unit-тесты для каждого правила;
- добавлен интеграционный тест анализатора через `analysistest`;
- подготовлен `testdata` с отдельным модулем и реальным `zap`.

## Структура проекта

- `cmd/loglint/main.go` — точка входа линтера;
- `internal/analyzer/analyzer.go` — запуск анализа и репорт диагностик;
- `internal/analyzer/extractor/` — извлечение текста лог-сообщения;
- `internal/analyzer/model/` — модель извлеченного сообщения;
- `internal/analyzer/rules/` — реализации правил (`lowercase`, `english`, `specials`, `sensitive`);
- `internal/analyzer/testdata/` — интеграционные тестовые фикстуры.

## Запуск

Запуск линтера как standalone-анализатора:

```bash
go run ./cmd/loglint ./...
```
